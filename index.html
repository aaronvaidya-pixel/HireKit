🔑 API Key Management

For production (recommended): Store one master Anthropic API key in your backend (e.g. a Vercel serverless function). Users never see or manage a key — calls go through your server. This is the proper SaaS model.

Current mode (browser-based): Each user pastes their own key. Stored locally in their browser. Sufficient for testing and personal use.

🚀 Deploy to Your Domain
Step 1 — Upload to Vercel (Free)

Go to vercel.com → Sign up free → Click "Add New Project" → Drag and drop this HTML file → Click Deploy. Done in 2 minutes.

Step 2 — Connect Your Domain

In Vercel → Project Settings → Domains → Add your domain (e.g. aaronvaidya.com). Vercel gives you DNS records to add at your domain registrar. Takes 5–10 minutes to go live.

Step 3 — For Production API Calls (Important)

Currently the app calls the Anthropic API directly from the browser. For a proper SaaS product, create a Vercel serverless function (/api/generate.js) that holds your master API key. Users call your server, not Anthropic directly. This hides your key and lets you control costs. Email contact@aaronvaidya.com if you need help setting this up.
